<pre>Точка входа: http://188.246.224.156/api/v1.1/reservations/add
Метод: POST
Обязательные параметры:
	branch_id=число //ID филиала
	comp_id=число //номер компьютера
	packet_id=число //ID тарифа
Необязательные параметры:
	user_id=число //ID пользователя (при отсутствии будет использован ID залогиненого пользователя)
	branch_id=число //ID филиала (при отсутствии будет получен при помощи IP адреса, с которого идет запрос) 
	start=строка //Момент начала резервации (при отсутствии будет использован момент, в который пришел запрос)
Возможные ошибки:
	ERROR_EMPTY_COMP_ID - не указан номер компьютера
	ERROR_REQUIRED_BRANCH_ID - нужно указать ID филиала
	ERROR_EMPTY_PACKET_ID - пустой ID тарифа
	ERROR_INVALID_PACKET - тариф не найден в базе данных
	ERROR_CROSSING_RESERVATIONS - время резервации пересекается с другой резервацией
	ERROR_BALANCE_NOT_ENOUGH - у пользователя не хватает баланса для покупки пакета
