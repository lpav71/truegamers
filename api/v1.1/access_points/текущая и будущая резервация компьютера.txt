<pre>Точка входа: http://188.246.224.156/api/v1.1/reservations/comp_reservations
Пример GET-запроса: http://188.246.224.156/api/v1.1/reservations/comp_reservations/?comp_id=1
Метод: GET / POST
Авторизация: Обязательна
Обязательные параметры:
	comp_id=число
Необязательные параметры
	user_id=число      //ID пользователя для которого нужно проверить резервации
	branch_id=число    //ID филиала (при отсутствии происходит попытка определить филиал по IP-адресу, с которого приходит запрос)
Возможные ошибки:
	ERROR_EMPTY_COMP_ID - не указан ID компьютера
	ERROR_REQUIRED_BRANCH_ID - не указан ID филиала и IP адрес запроса не принадлежит ни одному филиалу

В результате будет выведен JSON-массив, содержащий:

{
  "result":"RESULT_SUCCESS", - флаг успеха
  "user_id":45, - ID пользователя
  "comp_id":1, - ID компьютера
  "branch_id":1, - ID филиала
  "payload":
  {
    "current_reservation":
    {
      "current_user":null, - текущая резервация текущего пользователя
      "another_user": - текущая резервация другого пользователя
      {
        "start":"2021-01-16 11:00:00", - начало резервации
        "end":"2021-01-16 16:00:00" - окончание резервации
      }
    }
,
    "next_reservation":
    {
      "current_user":null, - будущая резервация текущего пользователя
      "another_user":null - будущая резервация другого пользователя
    }
  }
}
</pre>
